# Contributor Setup

This guide gets a new collaborator from `git clone` to a build running on a real device. If anything here is out of date or missing, please open an issue or PR.

> **Audience:** developers who want to build, run, and modify the app — not end users. End-user install instructions live in [INSTALLATION_GUIDE.md](INSTALLATION_GUIDE.md).

---

## 1. Required directory layout

The minimal setup is just **one** clone:

```
<your-parent-dir>/
└── wellbeing-mapper-app/   ← this repo
```

The app's `open_background_locator` dependency is now fetched from a pinned
commit on GitHub, so a fresh `flutter pub get` is all you need. **You only need
to clone other repos if you are actively developing them in parallel.**

Optional sibling clones (only if relevant to your work):

```
<your-parent-dir>/
├── wellbeing-mapper-app/
├── open-background-locator/    ← only if editing the location plugin (see §6)
└── wellbeing-mapper-server/    ← only if running a local server (see §7)
```

### Clone commands

```bash
mkdir -p <your-parent-dir>
cd <your-parent-dir>

git clone git@github.com:ActivitySpaceLab/wellbeing-mapper-app.git
```

That's it for a default build. The other repos are only needed for the
workflows in §6 and §7.

## 2. Toolchain

| Tool | Required version | Notes |
|---|---|---|
| Flutter | **3.27.1** (Dart 3.6.0) | Other versions will not compile. Use [FVM](https://fvm.app/). |
| Xcode | 15+ | Needed for iOS builds. iOS deployment target is 13.0. |
| Android Studio | latest | Or just the Android SDK + command-line tools. |
| CocoaPods | recent | `sudo gem install cocoapods` |
| Node.js | 18+ | Only needed if you run the local server. |

### Installing Flutter via FVM

```bash
brew install fvm   # or: dart pub global activate fvm
cd wellbeing-mapper-app
fvm use 3.27.1     # downloads if needed
fvm flutter --version  # confirm Flutter 3.27.1 / Dart 3.6.0
```

Every `flutter` command in this guide can be run as `fvm flutter`.

## 3. One-time per-clone setup

From inside `wellbeing-mapper-app/`:

```bash
fvm flutter pub get          # resolves dependencies, including the OBL path dep

cd ios && pod install && cd ..   # CocoaPods install for iOS

./sync-version.sh            # generates android/local.properties with the
                             # current version code/name from pubspec.yaml
```

### Files that are NOT in git (you provide them locally)

| File | What it is | How to create |
|---|---|---|
| `android/local.properties` | Android SDK path + Flutter version code/name | Auto-generated by `./sync-version.sh`. Edit `sdk.dir` and `flutter.sdk` to match your machine. |
| `android/key.properties` | Release signing keystore | Only needed if you produce **release** APKs. Ask the project owner. Not required for debug builds. |
| `.env` files | None at the moment | Currently no `.env` is required by the app. |
| `participant_codes.json` | Server-side list of valid codes | Only relevant in the **server** repo, not in the app. |

## 4. Verify your setup

```bash
fvm flutter doctor          # everything except a real device should be green
fvm flutter analyze         # should report only the pre-existing warnings
fvm flutter test            # runs unit tests (some are known-pre-existing-failing)
```

If `flutter pub get` fails to fetch `open_background_locator` from GitHub, check
that your SSH/HTTPS auth to GitHub works and that you have network access. The
pinned commit in `pubspec.yaml` must exist on the OBL repo's default branch.

## 5. Running on a device or simulator

### Android emulator or iOS simulator

```bash
fvm flutter run --dart-define=APP_FLAVOR=production
```

`APP_FLAVOR` can be `production` or `beta`. See [GIT_FLOW_BUILD_FLAVORS_GUIDE.md](GIT_FLOW_BUILD_FLAVORS_GUIDE.md) for the full flavor matrix.

### Real device

1. Plug the device in and trust the computer.
2. Enable Developer Mode (Android: Settings → About → tap Build Number 7×; iOS: Settings → Privacy → Developer Mode).
3. `fvm flutter run -d <device-id>` (find the id with `fvm flutter devices`).

### Production-vs-testing endpoint safety

The app reads the research server URL from `ENV.apiBaseUrl`, which defaults to a placeholder (`https://research-server.example.com/api/v1`). When the URL is the placeholder, **all upload code paths refuse to send anything** — debug builds cannot accidentally hit a real server.

To direct uploads somewhere real, pass `--dart-define`:

```bash
# Send to your locally-running wellbeing-mapper-server
fvm flutter run \
  --dart-define=APP_FLAVOR=production \
  --dart-define=SERVER_BASE_URL=http://localhost:3000/api/v1

# For Android emulator, use 10.0.2.2 instead of localhost:
fvm flutter run \
  --dart-define=APP_FLAVOR=production \
  --dart-define=SERVER_BASE_URL=http://10.0.2.2:3000/api/v1
```

Or just use the shortcut script (auto-picks the right host):

```bash
./scripts/run-with-local-server.sh
```

## 6. (Optional) Working on `open_background_locator` locally

If you need to edit OBL and see changes immediately in this app (without
pushing OBL and bumping the commit ref), use Dart's [`pubspec_overrides.yaml`](https://dart.dev/tools/pub/dependencies#dependency-overrides)
mechanism:

```bash
# 1. Clone OBL anywhere convenient.
cd <your-parent-dir>
git clone git@github.com:ActivitySpaceLab/open-background-locator.git

# 2. In wellbeing-mapper-app/, create pubspec_overrides.yaml
#    (this file is gitignored and only affects your local builds):
cat > pubspec_overrides.yaml <<'EOF'
dependency_overrides:
  open_background_locator:
    path: ../open-background-locator   # adjust to wherever you cloned it
EOF

# 3. Re-resolve
fvm flutter pub get
```

Flutter will now use your local OBL working copy. Delete
`pubspec_overrides.yaml` to go back to the pinned git ref.

### Promoting OBL changes back to the app

After pushing changes to OBL, bump the pin in this repo:

```bash
cd <path-to-open-background-locator>
git push                      # push your OBL changes
NEW_SHA=$(git rev-parse main)

cd <path-to-wellbeing-mapper-app>
sed -i '' "s/      ref: .*/      ref: ${NEW_SHA}/" pubspec.yaml
fvm flutter pub get           # updates pubspec.lock
git add pubspec.yaml pubspec.lock && git commit -m "Bump open_background_locator to ${NEW_SHA:0:7}"
```

## 7. (Optional) Running the local server

If you want to exercise the full encrypted-upload pipeline on your machine:

```bash
cd ../wellbeing-mapper-server   # sibling directory from section 1
./scripts/dev.sh                # bootstraps .env + node_modules, runs npm run dev
```

Server defaults: `http://localhost:3000`, storage in `./received/`. End-to-end smoke test: `./scripts/smoke-test.sh`.

See the [wellbeing-mapper-server README](https://github.com/ActivitySpaceLab/wellbeing-mapper-server) for full server docs.

## 8. Common gotchas

- **`flutter pub get` fails on `open_background_locator`** — either your GitHub
  auth/network is broken, or the pinned commit no longer exists on the OBL
  repo. Re-run after fixing network, or update the `ref:` in `pubspec.yaml`.
- **Local OBL edits not picked up** — you forgot to create
  `pubspec_overrides.yaml`. See §6.
- **iOS build fails on `audioplayers_darwin`** — your iOS deployment target is below 13.0. Run `pod install` inside `ios/` and verify the project sets `IPHONEOS_DEPLOYMENT_TARGET = 13.0`.
- **"flutter.sdk not set" on Android build** — `android/local.properties` is missing or wrong. Run `./sync-version.sh` and edit the file to point at your Flutter install.
- **Map shows red error overlay on extreme zoom-out** — should be fixed by the `minZoom`/`maxZoom` bounds, but if it recurs, file an issue with the device and zoom level.
- **APK won't upgrade an existing install** — version code didn't increment. Run `./sync-version.sh` and rebuild.
